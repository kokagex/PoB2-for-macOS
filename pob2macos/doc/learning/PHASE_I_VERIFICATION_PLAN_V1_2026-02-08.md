# Phase I 検証計画 V1 (2026-02-08)

## 前提
BuildOutput() が成功した状態。計算エンジンが動作中。
サイドバーに計算値が表示されている（Life:33, Mana:41, Resistance:-60%等）。

## 検証項目

### Step 1: パッシブノード割当 → Stats更新 (最重要)
**目的**: ツリーのノードをクリックして割り当て、サイドバーStatsが更新されるか確認
**方法**:
1. アプリ起動
2. パッシブツリーでノードをクリック（Rangerスタート付近）
3. サイドバーのLife/Dex等が変化するか確認
4. スクリーンショットで視覚的検証

**予想される問題**:
- ノードクリック → allocNode → BuildOutput再計算 の連鎖が動作するか
- buildFlag.specが設定されてBuildOutputがトリガーされるか

### Step 2: ツリーTooltip確認
**目的**: ノードホバー時にtooltipが表示されるか
**方法**: ノードの上にマウスを置いて確認
**予想される問題**:
- grantedPassivesデータの有無
- tooltip描画コードの動作

### Step 3: ヒートマップ確認
**目的**: "Show Node Power"ボタンでヒートマップ表示されるか
**方法**: 画面下部の"Show Node Power"ドロップダウンで確認
**予想される問題**:
- fullDPS等の計算結果が必要
- 色分け計算の動作

## 期待される結果
- ノード割当でStats更新 → Phase I成功
- Tooltip表示 → Phase I成功
- 問題がある場合 → 追加修正（反復的）

## 想定時間: 15分（検証10分 + 追加修正5分）

## リスク: 低（検証タスク、既存機能の確認のみ）
## ロールバック: 不要（検証のみ）
